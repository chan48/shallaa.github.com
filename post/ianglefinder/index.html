앱스토어

지금은 눈이 침침해서ㅋ 잘 안쓰지만 한참 Codea를 가지고 놀때 만들었던 앱이다.

원래는 자전거 타고 산을 올라가는 취미( 영어로 뭐라 하던데 기억이 안나네.. )가 있는 친구를 위해서 만든 어플이었는데 그 친구가 안드로이드 폰을 사용하면서 참 부질없는 어플이 되었다.


대충 동영상을 보면 감이 올텐데.. Codea는 아이패드에서 코딩 & 테스트를 하고 어플을 제작할 수 있게 해주는 어플이다.

어플 소개에 "It's kind of like Garage Band of coding." 라는 멘트가 있는데 이게 딱 맞는 표현인 것 같다.

지금도 가끔 간단한 프로토타입을 만들때나 스터디 중에 예제같은 것이 필요하면 유용하게 잘 쓰고 있다.

나처럼 출퇴근 시간이 한시간이 넘는 장거리라면 강추할만 하다. 버스에서 심심한 시간에 코드를 끄적거리면서 이것저것 만들다 보면 시간이 금방 잘간다.

데스크탑에서라면 더 좋은 Lua 툴들이 많다. Corona SDK도 있고, Gideros도 있고.. 개인적으로는 Corona SDK도 참 좋은 툴이지만 그보다는 Gideros를 더 추천.

이동중에는 Codea로 대충 작성을 하고 Gideros에 코드를 붙여넣기해서 활용하는 방법도 있다. 또는 Gideros도 아이패드에서 코딩 & 테스트를 할 수 있는 셋팅이 있는데.. 그건 나중에.

아이패드에서 코딩 & 테스트가 완료되면 Codea Runtime을 이용해서 어플로 제작을 할 수 있다.

어플로 제작을 하는 방법은 README만 살펴보아도 금방 알 수 있다.

1. Run the make_project.sh script from a Terminal session. It takes a parameter which is the name of the app.
2. Open the CodeaTemplate.xcodeproj project in Xcode.
3. Delete the existing the Project.codea file from the Classes group and select Move To Trash
4. Rename your codea project Project.codea
5. Drag and drop your project into the Xcode project
6. Check the "Copy items into destination folder's group (if needed)"
7. Select "Create folder references for any added folders" and make sure your app's target is selected. Click Finish
8. Setup the bundle id and Icon for your project as per usual

커뮤니티도 꽤 잘되어 있어서 Codea Forums을 보면 각종 플러그인이나 유용한 예제들이 참 많다.

API 업데이트도 꾸준히 잘되고 있고 Air Code라는 기능이 생겨서 데스크탑에서 코드를 작성하고 아이패드에서 테스트를 하는 것도 가능하다.

포럼을 보면 Codea만 사용하는 인디 개발자도 있는 모양인데 그 보다는 취미로 추천할만 하다.
