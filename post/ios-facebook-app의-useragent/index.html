iOS facebook app의 userAgent

Tweet

iOS의 facebook 앱에서 링크를 통해 웹페이지를 열 경우 facebook 앱 내부에서 페이지가 열리게 된다.

Android에서는 외부 브라우저로 열리는 것 같은데 ( 테스트 해 본 기기들에서는 전부 ) 모든 기기들을 테스트 해 본 건 아니라서 혹시 모르겠다.

iOS에서 링크를 통해 페이지가 열렸을때, 페이지에서는 navigator.userAgent 문자열을 통해서 iPad, iPhone 여부를 얻어오고 있었는데 항상 iPhone이라는 결과가 얻어지는 문제가 있었다.

그래서 facebook 앱에서 페이지가 열렸을 경우 navigator.userAgent를 찍어보니

// iPhone
Mozilla/5.0 (iPhone; CPU iPhone OS 7_0_3 like Mac OS X) AppleWebKit/537.51.1 (KHTML, like Gecko) Mobile/11B511 [FBAN/FBIOS;FBAV/6.7.2;FBBV/603804;FBDV/iPhone5,2;FBMD/iPhone;FBSN/iPhone OS;FBSV/7.0.3;FBSS/2; FBCR/KT;FBID/phone;FBLC/ko_KR;FBOP/5]

// iPad
Mozilla/5.0 (iPad; CPU OS 7_0_4 like Mac OS X) AppleWebKit/537.51.1 (KHTML, like Gecko) Mobile/11B554a [FBAN/FBIOS;FBAV/6.7.2;FBBV/603804;FBDV/iPad2,7;FBMD/iPad;FBSN/iPhone OS;FBSV/7.0.4;FBSS/1; FBCR/olleh;FBID/tablet;FBLC/ko_KR;FBOP/1]

이런 식으로 찍히고 있었다.

문제는 iPad의 navigator.userAgent 에도 "iPhone"이라는 문자열이 포함되어 있었던 것.

var i;
    agent = navigator.userAgent.toLowerCase();
if( agent.indexOf( i = 'iphone' ) > -1 || agent.indexOf( i = 'ipad' ) > -1 ){
                device = i == 'ipad' ? 'tablet' : 'mobile';

코드가 이런 식으로 되어 있었는데 iPad의 navigator.userAgent에 iphone이라는 문자열이 포함되어 있으니 항상 iphone에 걸리게 된다.

이 경우는 간단히

var i;
    agent = navigator.userAgent.toLowerCase();
if( agent.indexOf( i = 'ipad' ) > -1 || agent.indexOf( i = 'iphone' ) > -1 ){
                device = i == 'ipad' ? 'tablet' : 'mobile';

체크하는 문자열의 순서만 바꿔주면 해결.
